# 要件定義

## 概要

ピアノ楽譜をアップロードすると、Minecraft の音符ブロック(ノートブロック)+レッドストーン回路でその曲を演奏する装置の「組み立て設計書」をブラウザに表示する Web アプリを開発する。

ユーザーはこのブラウザ画面を見ながら、手元の Minecraft サーバーで**手作業で**装置を組む。サーバー側プラグイン・MOD は作らない(設計書を見て人間が組む、が本プロダクトの体験)。

## 確定済み要件

| # | 項目 | 決定内容 |
|---|---|---|
| 1 | 演奏装置 | **音符ブロック前提**。ジュークボックスはレコード再生専用で任意の音を鳴らせないため。「叩いて音程を変える(0〜24回)」は音符ブロックの仕様であり、ユーザーの意図と一致することを確認済み |
| 2 | 入力形式 | **MIDI (.mid) / MusicXML (.musicxml, .mxl) / PDF・画像楽譜** の3種すべて。PDF・画像は OMR(光学楽譜認識)で変換し、精度は実験的でよい(ユーザー了承済み) |
| 3 | 技術スタック | **Python + FastAPI**。楽譜パースに music21 を使用。フロントエンドはビルド不要の素の HTML/ES Modules |
| 4 | 起動方式 | **docker compose** で起動できること |
| 5 | プレビュー再生 | **あり**。ブラウザ上で音符ブロック風の合成音により、量子化後のタイミングを組む前に聴感確認できる(Minecraft 実音源は著作物のため同梱しない) |

## 設計書に必須の4点

すべてのステップ表示に以下が揃っていること(受け入れ基準):

- **(a)** 前ステップからの遅延と**リピーター構成**(例: 7 RT → リピーター2個: 4目盛+3目盛)
- **(b)** 音符ブロックの**下に置くブロック**の日本語名(例: 土、オークの板材、金ブロック)
- **(c)** 音符ブロックを**何回叩く**か(右クリック回数 0〜24)
- **(d)** **右手/左手**の別(左手パートがあるときは「これとこれを同時に鳴らす」が分かること)

## ユーザーの動作環境(参考)

| 項目 | 内容 |
|---|---|
| サーバー | Paper 26.2(experimental チャンネル)、バニラ挙動 |
| Java | 25(25.0.x で確認) |
| ビルド | JDK 25 + Maven |

音符ブロック・レッドストーンの挙動はバニラ準拠なので、本アプリはサーバーバージョンに依存しない。プラグイン連携は不要。

## スコープ外

- サーバープラグインによる自動建築・自動演奏
- Minecraft 実サウンドファイルの同梱(著作物)
- コマンド/ストラクチャブロック用ファイル(.schem 等)の出力(将来検討は可)
